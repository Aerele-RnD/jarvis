# Research: AI Assistant Mobile Apps, Mid-2026 — Field Study for the Jarvis Mobile App

> Supporting research for the Jarvis mobile app design (see `2026-07-06-mobile-backend-design.md`). Produced 2026-07-06 by a web-research agent; sources cited inline, unverified claims flagged.

**Scope & method.** Researched 2026-07-06 via parallel web-research passes over official help centers, release notes/changelogs, engineering job postings, platform docs, and tech press (2025–2026 sources preferred). Every claim is cited inline; single-sourced or undocumented items are marked **[unverified]**. Target application: Jarvis — a B2B mobile app where an AI agent operates a customer's Frappe/ERPNext site via chat, with multi-minute agent turns, completion push notifications, voice input, and approve/confirm flows for ERP writes.

---

## 1. ChatGPT (OpenAI)

**Push.** The richest completion-push system of any app. Scheduled Tasks run server-side and "notify you via push notification and email once the task is complete" ([help.openai.com](https://help.openai.com/en/articles/10291617-scheduled-tasks-in-chatgpt)); Deep Research (5–30 min runs) notifies on completion so you can step away ([openai.com](https://openai.com/index/introducing-deep-research/)); ChatGPT Agent pushes when done ([help.openai.com](https://help.openai.com/en/articles/11752874-chatgpt-agent)); and Codex-in-ChatGPT (mobile preview, May 2026) pushes both when a task **finishes** and when it **needs input/approval** — approvals actionable from the phone ([9to5Mac](https://9to5mac.com/2026/05/14/openai-brings-codex-control-to-chatgpt-for-iphone-and-android/), [MacRumors](https://www.macrumors.com/2026/05/15/openai-brings-codex-chatgpt-mobile-app/)). Codex notifications explicitly deep-link to the completed task (iOS 1.2026.132 changelog, [developers.openai.com/codex/changelog](https://developers.openai.com/codex/changelog)). Settings offer per-category Push/Email toggles ([guideflow tutorial](https://www.guideflow.com/tutorial/how-to-disable-email-notifications-for-task-updates-in-chatgpt)).

**Voice.** Composer dictation is server-side (Whisper-family) ([datastudios.org](https://www.datastudios.org/post/can-chatgpt-transcribe-audio-and-how-does-it-work)); Advanced Voice Mode is realtime speech-to-speech, merged **into the chat view** in Nov 2025 (live transcript + streaming visuals, revertible to separate mode) ([MacRumors](https://www.macrumors.com/2025/11/26/chatgpt-voice-mode-update-seamless-chat/)); opt-in "Background Conversations" keeps voice alive when locked ([MacRumors how-to](https://www.macrumors.com/how-to/use-chatgpt-mobile-app-background-chat/)).

**Auth.** Email/password, Google/Microsoft/Apple; passkeys are first-class and default-after-email once enrolled ([help.openai.com](https://help.openai.com/en/articles/20001039-passkeys-to-secure-your-openai-account)), with a May 2026 Yubico partnership and passwordless "Advanced Account Security" mode ([BiometricUpdate](https://www.biometricupdate.com/202605/openai-rolls-out-passkeys-for-chatgpt-partners-with-yubico)); mobile-app approval pushes serve as login verification ([help.openai.com](https://help.openai.com/en/articles/9889414-why-am-i-being-asked-to-verify-my-login)). SAML/OIDC SSO for Business/Enterprise; multi-account switching is web-only so far ([help.openai.com](https://help.openai.com/en/articles/20001068-use-multiple-accounts-with-account-switching)).

**Long tasks.** Deep Research shows an activity log + sources panel (no progress bar), a duration expectation up front, and since Feb 2026 an editable research plan before launch ([MacRumors](https://www.macrumors.com/2026/02/11/chatgpt-deep-research-mode-document-viewer/)). All long tasks run server-side — killing the app doesn't kill the task. Live Activities are used **only for voice**, not agent tasks ([X/nathanwchan](https://x.com/nathanwchan/status/1714042573545955833)).

**Stack.** Native: OpenAI hires for a "core **Swift** platform" and "core **Kotlin**-based platform" powering ChatGPT/Sora ([iOS role](https://openai.com/careers/ios-engineer-chatgpt-mobile-infrastructure-san-francisco/), [Android role](https://openai.com/careers/android-engineer-chatgpt-mobile-infrastructure-san-francisco/)) — **confirmed**.

**Workflow.** Agent mode "explicitly ask[s] for your permission before taking actions with real-world consequences," with watch-mode pauses in sensitive contexts ([openai.com](https://openai.com/index/introducing-chatgpt-agent/), [system card](https://cdn.openai.com/pdf/839e66fc-602c-48bf-81d3-b21eacc3459d/chatgpt_agent_system_card.pdf)). Lock-screen voice widget ([Tom's Guide](https://www.tomsguide.com/ai/chatgpt/i-added-the-chatgpt-voice-widget-to-my-iphone-lock-screen-for-quick-access-to-advanced-voice-heres-how)), Shortcuts action, no first-party watch app ([MacRumors](https://www.macrumors.com/how-to/get-chatgpt-on-apple-watch/)).

## 2. Claude (Anthropic)

**Push.** The most explicit **two-class notification model** shipped anywhere: Claude Code Remote Control's `/config` has exactly two toggles — "**Push when Claude decides**" (long task finished) and "**Push when actions required**" (permission prompts/questions); users can even prompt "notify me when the tests finish"; pushes are suppressed while you're at the connected terminal ([code.claude.com/docs/en/remote-control](https://code.claude.com/docs/en/remote-control)). Dispatch (phone→Desktop tasks) pushes "when a task is done or when Claude needs your go-ahead" ([support.claude.com](https://support.claude.com/en/articles/13947068)). Notably, a completion push for regular Research runs is **not documented anywhere official** **[unverified]** ([support.claude.com Research article](https://support.claude.com/en/articles/11088861-use-research-on-claude)).

**Voice.** Voice mode (beta May 27, 2025, all plans) is a turn-based STT→LLM→TTS pipeline (ElevenLabs TTS, server-side), not realtime speech-to-speech ([TechCrunch](https://techcrunch.com/2025/05/27/anthropic-launches-a-voice-mode-for-claude/), [Simon Willison](https://simonwillison.net/2025/May/31/using-voice-mode-on-claude-mobile-apps/)); hands-free default with an explicit **push-to-talk** option for noisy environments; 19 languages since June 2026 ([support.claude.com](https://support.claude.com/en/articles/11101966-use-voice-mode)). Dictation is separate, with a strong privacy posture: audio deleted after transcription, never used for training ([support.claude.com](https://support.claude.com/en/articles/10065434-using-dictation-on-claude-mobile)).

**Auth.** Passwordless: Google OAuth or email magic-link/OTP ([support.claude.com](https://support.claude.com/en/articles/13189465-log-in-to-your-claude-account)); SAML SSO via WorkOS for Team/Enterprise, tested across mobile ([support.claude.com](https://support.claude.com/en/articles/13132885-set-up-single-sign-on-sso)). No passkey login, but **Trusted Devices** uses passkey/biometric step-up (sign-in ≤18h, refreshed via Face ID/passkey) to gate Remote Control ([code.claude.com](https://code.claude.com/docs/en/remote-control)).

**Long tasks.** Advanced Research runs up to 45 min ([mlq.ai](https://mlq.ai/news/anthropic-adds-advanced-research-search-capability-to-claude/)); Claude Code on web+iOS (Oct 20, 2025) runs cloud sandbox sessions with parallel tasks ([Engadget](https://www.engadget.com/ai/anthropic-brings-claude-code-to-ios-and-the-web-180023611.html)); Remote Control (Feb 25, 2026) mirrors a local session to the phone's "Code" tab — the local agent makes **outbound-only HTTPS, registers with the API and polls for work**, surviving laptop sleep/network drops ([code.claude.com](https://code.claude.com/docs/en/remote-control), [Simon Willison](https://simonwillison.net/2026/Feb/25/claude-code-remote-control/)).

**Stack.** The widely-repeated "Claude app is React Native/Expo" claim has **no citable primary source**: Claude is absent from [expo.dev/customers](https://expo.dev/customers), [reactnative.dev/showcase](https://reactnative.dev/showcase), and Expo's own "who's using Expo" lists ([evanbacon.dev](https://evanbacon.dev/blog/expo-apps)), while Anthropic's Staff iOS posting requires Swift/UIKit/SwiftUI ([Greenhouse](https://job-boards.greenhouse.io/anthropic/jobs/4572744008)). Treat RN/Expo as **[unverified rumor]**; native-first hiring is what's confirmed.

**Workflow.** Claude Code **permission prompts arrive on the phone** — "you decide what runs" ([code.claude.com](https://code.claude.com/docs/en/remote-control)) — the closest shipped analog to Jarvis's ERP write-approvals. iOS widget includes one-tap "open in dictation mode"; "Analyze Photo" works from Lock Screen without unlocking ([support.claude.com](https://support.claude.com/en/articles/10263469-using-claude-app-intents-shortcuts-and-widgets-on-ios)). No watch app.

## Takeaways for the Jarvis mobile app

1. **Adopt the universal server-side pattern: run the turn on the server, push on completion, deep-link, re-fetch.** No app streams agent progress in the background — iOS background URLSession can't hold SSE and Android pushes apps toward FCM ([Apple docs](https://developer.apple.com/documentation/Foundation/downloading-files-in-the-background), [Android docs](https://developer.android.com/develop/background-work/background-tasks/data-transfer-options)). Jarvis agent turns must be fully owned by the fleet/server (already true architecturally); the app is a viewer that drops its stream on background and re-fetches conversation state on open or on notification tap.

2. **Implement resumable streams with an event cursor.** OpenAI's background mode is the canonical design: `background: true`, per-event `sequence_number`, resume via `starting_after=<cursor>` ([developers.openai.com](https://developers.openai.com/api/docs/guides/background)). Give the Jarvis stream a monotonically numbered event log per turn so a killed/backgrounded app replays exactly what it missed instead of re-rendering from scratch. Anthropic's fallback for >10-min work — polling — is a fine degraded mode ([Anthropic streaming docs](https://platform.claude.com/docs/en/build-with-claude/streaming)).

3. **Ship two notification classes from day one: "task finished" and "Jarvis needs your decision."** This is exactly Claude Code's two-toggle model ([code.claude.com](https://code.claude.com/docs/en/remote-control)) and Codex's finish-or-needs-input pushes ([MacRumors](https://www.macrumors.com/2026/05/15/openai-brings-codex-chatgpt-mobile-app/)). For Jarvis: separate APNs/FCM channels so a user can silence "done" pings but never miss "approve this Sales Invoice posting." Make the approval class time-sensitive (iOS) / high-priority (Android).

4. **Make approval pushes actionable, and gate them with biometric step-up.** Codex approval requests are actionable from the notification ([laozhang teardown](https://blog.laozhang.ai/en/posts/openai-codex-mobile-app)); Claude's Trusted Devices requires recent auth refreshed via Face ID/passkey before remote actions ([code.claude.com](https://code.claude.com/docs/en/remote-control)). For ERP writes, put Approve/View actions on the notification, but require device biometric (LocalAuthentication/BiometricPrompt) before the approve commits — sign the approval with a device-bound key for auditability. *(Design note: platform verification later showed the commit itself must happen in a foregrounded app — see the backend design's changelog.)*

5. **Confirmation UI = show the exact write, editable before commit.** Gemini's message-send preview (recipient + full text, revisable before sending) is the gold standard ([support.google.com](https://support.google.com/gemini/answer/15574928)); ChatGPT Agent asks before "real-world consequences" ([openai.com](https://openai.com/index/introducing-chatgpt-agent/)). Jarvis approvals should render the DocType diff (doc name, fields changed, amounts, submit-vs-save) — never a bare "Jarvis wants to proceed. OK?". MCP elicitation's three-action model (accept / decline / cancel) plus "show which server/tool is asking" is the right protocol shape ([modelcontextprotocol.io](https://modelcontextprotocol.io/specification/2025-06-18/client/elicitation)).

6. **Progress UI: activity log + expectation-setting, not a progress bar.** Every deep-research product shows a narrated activity feed ("reading Sales Orders… checking stock ledger") with an upfront duration hint and "you can leave this chat" affordance ([OpenAI](https://openai.com/index/introducing-deep-research/), [Google](https://support.google.com/gemini/answer/15719111), [Mistral](https://docs.mistral.ai/le-chat/research-analysis/deep-research)). Jarvis should surface tool-call names as human-readable steps and let users navigate away mid-turn — the turn belongs to the thread, not the screen.

7. **For expensive multi-step ERP jobs, use plan-then-approve.** Gemini and ChatGPT both added an editable plan the user approves before the long run starts ([support.google.com](https://support.google.com/gemini/answer/15719111), [MacRumors](https://www.macrumors.com/2026/02/11/chatgpt-deep-research-mode-document-viewer/)). For batch ERP operations ("reconcile these 40 payments"), have Jarvis present the action plan once, collect one approval, then run — instead of 40 interrupting confirmations.

8. **Add a task inbox grouped by status.** Perplexity's Tasks view — **Scheduled / Needs attention / Completed**, per-task pause/cancel, per-task notification channels, and pushes that deep-link to the task session ([help center](https://www.perplexity.ai/help-center/en/articles/11521526-perplexity-tasks)) — is the most complete model. A B2B Jarvis needs this more than consumer apps do: pending approvals are literally a work queue.

9. **Voice v1 = push-to-talk dictation with an editable transcript and Autosend toggle; realtime voice later.** Every app separates composer dictation from voice mode; Le Chat's Voxtral PTT + Autosend ([docs.mistral.ai](https://docs.mistral.ai/le-chat/conversation/voice-mode)) and Claude's PTT-for-noisy-environments option ([support.claude.com](https://support.claude.com/en/articles/11101966-use-voice-mode)) fit ERP contexts (warehouses, shop floors). All majors do ASR server-side, but Jarvis can differentiate for B2B privacy by using on-device iOS Speech/Android SpeechRecognizer — and can cite Claude's "audio deleted after transcription" posture as the bar for any server path ([support.claude.com](https://support.claude.com/en/articles/10065434-using-dictation-on-claude-mobile)).

10. **Auth: site-first login → system-browser OIDC → long-lived session, passwordless by default.** For single-tenant Frappe sites, copy Mistral's domain-detection flow (enter site/work email → button becomes "Sign in with SSO") ([docs.mistral.ai](https://docs.mistral.ai/admin/security-access/sso)): Jarvis screen 1 asks for the site URL (or scans a QR from the admin console — Claude's Remote Control QR pairing is the pattern: [code.claude.com](https://code.claude.com/docs/en/remote-control)), then runs Frappe's OAuth2/OIDC in ASWebAuthenticationSession/Custom Tabs. Avoid in-app passwords entirely; note Perplexity's SP-initiated-only SSO constraint as the simplification precedent. Passkeys are table stakes trajectory (OpenAI, Microsoft, Google all default to them) — plan for them even if v1 uses OIDC.

11. **Engineer notification delivery hygiene explicitly.** Claude's docs warn iOS Focus modes and Android battery optimization delay pushes, and expose a "No mobile registered" self-check ([code.claude.com](https://code.claude.com/docs/en/remote-control)); ChatGPT/Perplexity pair push with **email fallback** for unseen results ([help.openai.com](https://help.openai.com/en/articles/10291617-scheduled-tasks-in-chatgpt), [Perplexity changelog](https://www.perplexity.ai/changelog/april-2025-product-update)); Claude Code suppresses pushes while you're present at the terminal ([code.claude.com](https://code.claude.com/docs/en/remote-control)). Jarvis: send push + email for approvals, suppress pushes when the app is foregrounded on that conversation, and add an in-app notification-health check.

12. **The whitespace: live progress on the lock screen.** No AI assistant uses iOS Live Activities or Android 16 `ProgressStyle`/Live Updates for agent-task progress — ChatGPT's only Live Activity is voice mode ([MacRumors](https://www.macrumors.com/how-to/use-chatgpt-mobile-app-background-chat/), [Android 16 docs](https://developer.android.com/about/versions/16/features/progress-centric-notifications)). ERP jobs are user-initiated start-to-end journeys — precisely the intended use case. "Posting invoices… 12/45 done" in the Dynamic Island would be a genuine differentiator no major assistant ships today.

13. **Stack choice: React Native/Expo is validated for exactly Jarvis's situation.** Frontier labs with huge teams build native (OpenAI, Google, Perplexity, xAI — all confirmed via job postings above), but Mistral shipped a 1M-downloads-in-two-weeks app on React Native with a small team ([job posting](https://www.jointaro.com/jobs/mistral-ai/staff-mobile-engineer-react-native-le-chat/)); the popular "Claude is RN/Expo" claim remains unverified. For a small B2B team, RN/Expo (or Flutter) is the pragmatic call — the features that matter (FCM/APNs, deep links, biometric step-up, background re-fetch, streaming with cursor resume) are all first-class there. Whatever the framework, budget for the native edges: notification actions, Live Activities (native module), and app-attestation if approvals are signed on-device.
